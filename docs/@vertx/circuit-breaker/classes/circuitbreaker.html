executeAndReportWithFallback

  • executeAndReportWithFallback(resultFuture: Future, command: function, fallback: function): CircuitBreaker
    • Defined in home/plopes/Projects/reactiverse/es4x/generator/io.vertx/vertx-circuit-breaker/npm/index.d.ts:144

    Executes the given operation with the circuit breaker control. The operation is generally calling an external system. The operation receives a {@link Future} object as parameter and must call {@link Future#complete(Object)} when the operation has terminated successfully. The operation must also call {@link Future#fail(Throwable)} in case of failure.

    The operation is not invoked if the circuit breaker is open, and the given fallback is called immediately. The circuit breaker also monitor the completion of the operation before a configure timeout. The operation is considered as failed if it does not terminate in time.

    Unlike {@link #executeWithFallback(Handler, Function)}, this method does return a {@link Future} object, but let the caller pass a {@link Future} object on which the result is reported. If the fallback is called, the future is successfully completed with the value returned by the fallback function. If the fallback throws an exception, the future is marked as failed.

    Executes the given operation with the circuit breaker control. The operation is generally calling an external system. The operation receives a {@link Future} object as parameter and must call {@link Future#complete(Object)} when the operation has terminated successfully. The operation must also call {@link Future#fail(Throwable)} in case of failure.
