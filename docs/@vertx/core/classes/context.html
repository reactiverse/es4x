executeBlocking

  • executeBlocking(blockingCodeHandler: function, ordered: boolean, resultHandler: function): void
  • executeBlocking(blockingCodeHandler: function, resultHandler: function): void
    • Defined in home/plopes/Projects/reactiverse/es4x/generator/io.vertx/vertx-core/npm/index.d.ts:1473

    Safely execute some blocking code.

    Executes the blocking code in the handler {@code blockingCodeHandler} using a thread from the worker pool.

    When the code is complete the handler {@code resultHandler} will be called with the result on the original context (e.g. on the original event loop of the caller).

    A {@code Future} instance is passed into {@code blockingCodeHandler}. When the blocking code successfully completes, the handler should call the {@link Future#complete} or {@link Future#complete(Object)} method, or the {@link Future#fail} method if it failed.

    Parameters

    • blockingCodeHandler: function

      handler representing the blocking code to run

        • (result: Future): void

        • Parameters

          • result: Future

          Returns void

    • ordered: boolean

      if true then if executeBlocking is called several times on the same context, the executions for that context will be executed serially, not in parallel. if false then they will be no ordering guarantees

    • resultHandler: function

      handler that will be called when the blocking code is complete
