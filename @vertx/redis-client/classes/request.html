Class Request

Builder for REDIS requests that will be encoded according to the RESP protocol was introduced in Redis 1.2. Which became the standard way for talking with the Redis server in Redis 2.0.

Redis protocol documentation states:

Clients send commands to a Redis server as a RESP Array of Bulk Strings.

So all non String/Bulk types will be encoded to Bulk for convenience.

    Adds a null encoded string. Redis does not allow storing the null value by itself. This method will encode any null value as the four character long string "null".

    As a recommendation, this method should not be used directly unless this is the intented behavior. It is present to handle special cases such as encoding of and which may contain null values.
